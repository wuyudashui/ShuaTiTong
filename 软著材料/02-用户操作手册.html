1. 软件概述

刷题通是一款面向学生和编程学习者的交互式刷题 Web 应用程序。支持单选、判断、多选、填空四种题型，提供 AI 智能解析和错题自动收集功能，帮助用户高效巩固知识。

软件为纯前端应用，所有数据存储在用户本地浏览器中，无需注册账号，无需后端服务器。支持通过 Capacitor 打包为 Android APK 安装包，也可直接通过浏览器使用。

1.1 核心功能

  • 题库上传：通过点击选择或拖拽方式上传 JSON 格式题库文件
  • 四种题型刷题：单选、判断、多选、填空
  • 题型筛选：按题型分类刷题
  • 模拟考试：按题型分组随机抽题，选项打乱，统一判卷
  • AI 智能解析：调用大语言模型自动生成题目解析
  • 错题本：自动收集答错题目，支持一键跳转复习
  • 答题统计：实时显示正确数、错误数、正确率
  • 暗色模式：护眼暗色主题切换
  • 进度持久化：自动保存答题进度，刷新不丢失
  • 键盘快捷键：← 上一题、→ 下一题、空格随机

1.2 技术架构

  • 前端框架：原生 TypeScript + Vite
  • 数据持久化：localStorage
  • AI 接口：DeepSeek API（浏览器直接调用）
  • 移动端打包：Capacitor v7（Android）
  • 状态管理：发布-订阅模式（Store 类）
  • 渲染策略：QuestionRenderer 接口（策略模式）

4. 快速开始

4.1 准备题库文件

题库文件为 JSON 格式，结构如下：

[
  {
    "id": 1,
    "type": "single",
    "question": "题干文本",
    "options": { "A": "选项A", "B": "选项B", "C": "选项C", "D": "选项D" },
    "answer": "B",
    "difficulty": "易",
    "explanation": "解析内容（可为空）"
  }
]

支持字段说明：

字段必填说明
id是题目唯一编号（数字）
type是题型：single（单选）、judge（判断）、multi（多选）、fill（填空）
question是题干内容
options是选项/填空，键值对格式
answer是正确答案。单选/判断填字母；多选填字母组合（如"ACD"）；填空留空
difficulty否难度：易、中、难
explanation否预先准备的解析文本

4.2 上传题库

  1. 点击"📂 上传 JSON 题库"按钮，选择 JSON 文件
  2. 或将 JSON 文件拖拽到上传区域
  3. 上传成功后，页面将显示题目数量和答题统计

图 4-1：题库上传界面

4.3 开始答题

上传题库后，默认显示第一道题目。点击选项即可作答，系统会自动判分并显示解析。使用底部按钮切换题目。

5. 功能介绍

5.1 题库上传

点击上传：点击"📂 上传 JSON 题库"按钮，从文件管理器中选择 JSON 文件。上传后自动解析并加载题目。点击"📁 最近打开"可以快速重新加载已上传过的题库文件。

拖拽上传：将 JSON 文件拖拽到虚线框区域，松开即可自动上传。拖拽时上传区域会高亮反馈。

格式模板：点击"📋 JSON 格式模版"可展开查看完整的题库格式示例，方便用户按模板制作自己的题库。

图 5-1：最近文件列表与格式模板

5.2 刷题模式

答题卡片上方显示题型标签和题目难度。点击选项即可作答，选择后系统立即判分：

  • 回答正确：选项边框变绿，反馈区显示"✅ 回答正确！"
  • 回答错误：选项边框变红，正确答案高亮显示，自动加入错题本

底部导航栏提供以下按钮：

  • ◀ 上一题：切换到上一道题目
  • 🎲 随机：随机跳转一道题目
  • 🤖 AI 解析：调用 AI 生成当前题目的详细解析（需配置 API）
  • 💡 显示答案：直接显示正确答案（此题会计入错误统计）
  • 下一题 ▶：切换到下一道题目

图 5-2：答题界面

页面顶部的统计栏实时显示：进度（当前题号/总题数）、正确数（绿色）、错误数（红色）、正确率。

5.3 四种题型

（一）单选题

点击选项字母或文字区域选择答案，选择后自动判分。选项左侧以圆形字母（A/B/C/D）标识。

图 5-3：单选题作答

（二）判断题

判断是对错题，选项固定为"正确"和"错误"（即 A/B 选项），点击即自动判分。逻辑与单选题相同。

（三）多选题

多选题支持选择多个选项，点击选项进行切换选择（选中项左侧复选框变为蓝色）。选择完成后点击"✓ 提交答案"按钮提交。判分时选项顺序不影响判分结果。

图 5-4：多选题作答

（四）填空题

填空题在文本框中输入答案，每个空对应一个输入框。填写完成后点击"✓ 提交答案"按钮。系统自动进行大小写不敏感匹配判分。答对的空变绿，答错的变红，同时显示正确答案。

图 5-5：填空题作答

5.4 题型筛选

在题型筛选栏中，点击对应标签可筛选特定题型的题目：

  • 全部：显示所有题型
  • 单选：仅显示单选题
  • 多选：仅显示多选题
  • 判断：仅显示判断题
  • 填空：仅显示填空题

筛选后题目导航和统计自动更新为当前筛选范围内的数据。切换筛选条件时，当前题号重置为第 1 题。

图 5-6：题型筛选栏

5.5 AI 解析

配置 AI 设置

点击顶部工具栏的"⚙️"按钮打开 AI 设置弹窗。配置以下参数：

  • API 地址：DeepSeek API 地址（默认：https://api.deepseek.com/v1）
  • 模型名称：选择或自定义模型（如 deepseek-v4-flash）
  • API Key：输入 DeepSeek API Key

设置说明：API Key 仅存储在你本机的浏览器 localStorage 中，不会上传到任何第三方服务器。请求由浏览器直接发送至 DeepSeek API。

图 5-7：AI 解析设置界面

使用 AI 解析

在任意题目下点击"🤖 AI 解析"按钮，系统将自动调用大模型生成题目的详细解析。解析内容包括：正确答案解释、各选项分析（针对选择题）、相关知识点说明。解析结果以富文本形式展示，支持代码高亮、列表、引用等格式。AI 解析获得的题目说明会自动保存，下次查看同一题时可直接显示。

图 5-8：AI 解析结果展示

5.6 错题本

点击顶部标签栏的"❌ 错题本"切换至错题本视图。错题本自动收集所有答错的题目，显示每道错题的题型、难度和正确答案。

功能特点：

  • 自动收集：答错的题目自动加入错题本
  • 数量统计：标签栏中实时显示错题数量
  • 点击跳转：点击任意错题项，自动跳转到该题目所在的刷题位置
  • 清空机制：在刷题模式下回答正确后，对应题目自动从错题本移除

图 5-9：错题本视图

5.7 暗色模式

点击顶部工具栏的"🌙"按钮可切换暗色/亮色模式。暗色模式采用深色背景（深蓝色系），降低屏幕亮度，适合在夜间或弱光环境下使用。切换后所有界面元素（包括统计栏、题目卡片、反馈区等）都会自动适配暗色主题。主题选择会自动保存，下次打开时保持上次的选择。

图 5-10：暗色模式界面

5.8 键盘快捷键

快捷键功能
←（左箭头）上一题
→（右箭头）下一题
空格键随机跳转一道题目

注意：当输入框或文本编辑框获得焦点时，快捷键不会触发，避免干扰文字输入。

5.9 模拟考试

模拟考试功能为用户提供接近真实考试的刷题体验。在考试模式下，系统会按题型分组，用户可选择每种题型抽取的题目数量，组卷后进行限时或不限时作答，最后统一提交判卷。

5.9.1 开始模拟考试

点击顶部工具栏的"📝 模拟考试"按钮，弹出考试设置弹窗。弹窗中按题型（单选、判断、多选、填空）列出题库中可用的题目数量。用户可调整每种题型希望抽取的题目数量，默认为该题型全部题目。点击"📋 全选"按钮可一键将所有题型设置为最大值。设置完成后点击"📝 开始考试"进入考试模式。

图 5-11：模拟考试设置界面

5.9.2 考试模式界面

进入考试模式后，界面会隐藏练习模式的标签栏和筛选栏，呈现干净的考试视图。顶部显示考试横幅（已答/总题数），题型标签页（如"单选（5题）""判断（3题）"等）可点击快速跳转到对应题型区域。每道题目的选项顺序会被随机打乱，防止记忆选项位置。

图 5-12：考试模式答题界面

5.9.3 答题与提交

在考试模式下，选择答案后不会立即显示对错，也不显示解析。用户可自由切换题目并修改答案。点击底部的"📝 提交试卷"按钮可提前交卷（无需答完所有题目）。提交后会显示成绩单，包含总分、正确数/错误数、正确率、各题型得分明细。

图 5-13：考试成绩单

5.9.4 答题回顾

在成绩单页面，点击"📋 逐题查看"按钮进入答题回顾模式。在此模式下，用户可以逐题查看自己的答案和正确答案对比，并查看题目解析。回顾模式中选项不再可点击修改。

6. 数据管理

6.1 进度自动保存

所有答题进度（当前题目位置、正确/错误计数、错题本、答题记录）会自动保存到浏览器的 localStorage 中。关闭页面或重启浏览器后，重新打开软件即可恢复之前的答题进度。

6.2 手动重置进度

点击顶部工具栏的"🔄 重置"按钮，在弹出的确认框中点击"确定"后，将清除所有答题进度数据，包括正确/错误计数、答题记录和错题本。题库文件不会被删除。

6.3 数据清除

如需彻底清除所有数据（包括题库文件和 AI 设置），可通过浏览器设置清除该网站的 localStorage 数据。

7. 常见问题

Q1：上传题库后没有显示题目？

请检查 JSON 格式是否正确，确保是一个数组（以 [ 开头），且每道题包含必需的字段（id、type、question、options、answer）。可参考页面上的 JSON 格式模板。

Q2：AI 解析无法使用？

请确认已在设置中正确填写 API 地址和 API Key。DeepSeek API Key 可前往 platform.deepseek.com 获取。如果使用本地模型（如 Ollama），API 地址需填写本地地址，API Key 可留空。

Q3：答题进度丢失？

请确认浏览器已启用 localStorage（默认开启）。清除浏览器缓存或 cookies 可能会导致数据丢失。建议定期导出题库以防丢失。

Q4：如何将软件安装到手机？

可通过 Capacitor 打包为 Android APK 安装包。在项目目录下依次执行 npm run build、npx cap sync android，然后在 Android Studio 中构建 APK。

Q5：支持哪些题型？

支持四种题型：单选（single）、判断（judge）、多选（multi）、填空（fill）。每种题型的作答方式和判分逻辑有所区别，详见第 5.3 节。

Q6：如何清除错题本？

在刷题模式下，答对的题目会自动从错题本移除。如需一次性清除所有错题，可使用"重置"功能，但这也会清除其他答题进度数据。

Q7：软件需要网络连接吗？

基础刷题功能完全离线可用。仅 AI 解析功能需要网络连接以调用远程 API。

Q8：模拟考试和普通刷题有什么区别？

模拟考试模式下，系统按题型分组随机抽题，选项顺序打乱，答题时不显示对错和解析，所有题目答完后统一提交判卷，生成成绩单。普通刷题模式则答题后立即判分显示解析。
